About Nova

Nova exists because long-running work deserves continuity.

Nova is being built around a simple problem: serious work keeps moving, but the context around that work is constantly being scattered across conversations, files, decisions, notes, and half-finished next steps.
Request Access Explore Features

The problem

Good work loses its thread.

A project may last weeks or months while important context gets split between chats, files, old decisions, changing priorities, and the question of what should happen next.

The idea

Keep the project closer to the conversation.

Nova connects ongoing conversation with project state, durable memory, sessions, and working material so more of the actual project can remain useful as the work changes.

The direction

Context should improve the next move.

Remembering information is only part of the goal. Nova is being shaped to use that context to help clarify priorities, organize work, and surface a more useful next action.

Why Nova

Built for work that is still becoming something.

Nova is not centered on producing one isolated answer and forgetting the path that led there. The product direction is focused on projects where ideas change, files matter, decisions accumulate, and yesterday's progress should still mean something tomorrow.
Continuity Keep the active project thread easier to continue.
Clarity Separate the current objective from old noise.
Judgment Use project context to make the next action more useful.
Control Keep project information, files, and work easier to inspect.
