Nova Help

Give Nova the thread. Then keep moving.

Nova works best when it understands the project, the current state, and what matters next. Start with the real work, add useful context, and continue the conversation.
Open Nova Contact Nova

01 ? Start with the project

Tell Nova what you are working on.

Explain the real project, the current problem, or the result you are trying to reach. Useful project details beat a giant ceremonial prompt.
I'm building a project and the current problem is...

02 ? Keep useful context

Say what should remain important.

Tell Nova when a decision, preference, constraint, or project fact should continue to matter across the work.
Remember that this is the direction we chose.

03 ? Work with files

Attach the material the task depends on.

Add a file or image when the answer depends on its contents. Ask Nova directly to summarize, inspect, explain, compare, or use it as project context.
Summarize this file and tell me what matters for the project.

04 ? Continue across sessions

Keep related work in a connected thread.

Stay in the same conversation when continuity matters. Start a new session when the project or purpose genuinely changes.
What were we working on, and what is the next move?

05 ? Ask for a plan

Turn a larger goal into practical steps.

Ask Nova to plan, build, fix, or repair a goal when the work has multiple steps. Review the direction and keep moving through the same project thread.
Auto-plan the safest way to finish this feature.

06 ? Continue naturally

Short continuation commands are enough.

When Nova already understands the active task, short commands can continue the work without repeating the full goal.
k ok next continue run step

A simple working rhythm

Context first. Judgment second. Action third.

Nova is most useful when the conversation stays connected to the project instead of becoming a pile of isolated prompts.
1 Describe the project and current problem.
2 Add the decision, file, or constraint that matters.
3 Ask for the next move, plan, fix, or explanation.
4 Continue the thread and correct Nova when project reality changes.

Still stuck?

Describe what you expected and what happened instead.

Include the feature you were using, the task you gave Nova, and the result that felt wrong. A clear example makes the problem much easier to understand.
