Nova Features

Keep the project connected.

Nova brings project context, sessions, memory, files, planning, and next actions into one workspace built for work that does not fit inside a single prompt.
Request Access View Roadmap

01 ? Continuity

Continue the same project.

Saved sessions help keep long-running work connected so the next conversation can continue from an existing thread instead of pretending the project began five minutes ago.

02 ? Project state

Keep the current state in view.

Nova can use project context to stay oriented around what has been completed, what is protected, what is blocked, and what matters now.

03 ? Memory

Retain the durable details.

Important preferences, project facts, checkpoints, and decisions can remain useful without turning every temporary debug line into permanent memory.

04 ? Files

Bring working material into the thread.

Uploads and attachments give Nova more of the actual project material needed to discuss code, text, images, and other working files with better context.

05 ? Planning

Turn a goal into a clearer path.

Planning tools help separate the objective, the current step, and the work still ahead so large tasks are easier to reason about and move forward.

06 ? Next actions

Focus on the next useful move.

Nova is being built to judge the active project state and surface a more useful next action instead of burying the user under another wall of generic suggestions.

One working thread

Context should lead somewhere.

The pieces are designed to work together. Conversation provides the active thread. Project state gives it direction. Memory protects durable context. Files add working material. Planning and next actions help turn that context into movement.
