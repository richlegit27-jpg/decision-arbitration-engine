Nova Roadmap

Turning a working prototype into a real AI workspace.

Nova is being built in layers: stabilize the desktop workspace, polish the public product, strengthen billing and memory, then rebuild mobile cleanly instead of dragging old frontend debt forward.
Now

Stabilize and present

The current focus is making Nova feel like a real product: public pages, stable desktop entry, billing dashboard, and clear positioning.

  • Desktop workspace stays protected
  • Landing, About, Features, Blog, Billing
  • Project memory and sessions stay core
  • Railway preview routes stay clean
Next

Make account controls real

The next layer is making billing, credits, user plans, usage, and model access easier to understand and safer to enforce.

  • Cleaner billing dashboard
  • Usage ledger visibility
  • Plan tiers and credit costs
  • Backend-enforced model allowlist
Later

Rebuild mobile cleanly

Mobile should become a simpler shell based on the stable product, not a pile of old experimental scripts fighting each other.

  • Separate mobile branch
  • Minimal script stack
  • Desktop feature parity where useful
  • Reliable upload/session flow

Product direction

The foundation is already there.

Nova already has the pieces of a serious AI workspace: chat, memory, sessions, uploads, voice tools, billing credits, deployment, and a stable desktop lane. The roadmap is about shaping those pieces into a clean product.
