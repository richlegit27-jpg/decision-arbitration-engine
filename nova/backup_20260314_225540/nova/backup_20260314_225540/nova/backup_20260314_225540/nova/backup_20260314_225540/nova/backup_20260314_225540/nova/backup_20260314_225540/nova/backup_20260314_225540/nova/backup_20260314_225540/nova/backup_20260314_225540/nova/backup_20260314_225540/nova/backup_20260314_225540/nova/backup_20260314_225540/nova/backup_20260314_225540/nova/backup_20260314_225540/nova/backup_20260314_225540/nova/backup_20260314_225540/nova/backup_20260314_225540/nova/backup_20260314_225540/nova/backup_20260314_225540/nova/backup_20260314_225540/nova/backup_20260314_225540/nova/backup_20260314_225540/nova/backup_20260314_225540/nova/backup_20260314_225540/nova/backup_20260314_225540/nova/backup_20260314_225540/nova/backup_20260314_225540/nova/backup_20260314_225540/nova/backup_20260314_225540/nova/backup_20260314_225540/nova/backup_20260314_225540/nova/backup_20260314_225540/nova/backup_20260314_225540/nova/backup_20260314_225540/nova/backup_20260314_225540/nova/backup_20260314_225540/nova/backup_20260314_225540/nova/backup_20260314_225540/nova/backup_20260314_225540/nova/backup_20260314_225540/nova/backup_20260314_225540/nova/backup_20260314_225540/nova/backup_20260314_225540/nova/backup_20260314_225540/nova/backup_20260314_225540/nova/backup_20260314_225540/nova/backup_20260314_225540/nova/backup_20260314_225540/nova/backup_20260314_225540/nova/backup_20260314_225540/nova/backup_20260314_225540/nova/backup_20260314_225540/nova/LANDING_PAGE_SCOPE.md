\# Nova Landing Page Scope



\## Goal

Create a clean public landing page for Nova that feels modern, clear, and product-ready.



\## Main Objective

When someone opens Nova, they should immediately understand:

\- what it is

\- why it matters

\- what it can do

\- what to click next



\## Landing Page Sections



\### 1. Hero Section

\- Nova name

\- one strong headline

\- one short subheadline

\- primary button

\- secondary button



\### 2. Product Summary

Short explanation of Nova as a local AI workspace with memory, attachments, and persistent interaction.



\### 3. Feature Section

Show the core features:

\- chat

\- memory

\- attachments

\- workspace awareness

\- local control



\### 4. How It Works

Simple explanation of the user experience:

\- open Nova

\- chat with Nova

\- keep context

\- work with files

\- continue across sessions



\### 5. Call To Action

Buttons for:

\- Get Started

\- Login



\### 6. Footer

Simple footer with product name and version or status



\## Visual Direction

\- clean

\- modern

\- dark polished look

\- strong spacing

\- clear text hierarchy

\- simple call-to-action buttons



\## Rule

The landing page should not feel cluttered or overloaded.

It should feel clear, sharp, and real.



\## Immediate Outcome

Nova should feel like a real product before the user even logs in.
